### 2. 租房价格核验

当用户提供房源价格信息时，执行以下**实时核验**流程：

1. 提取用户提供的城市、区域、面积、户型、租金
2. **实时获取最新租房价格数据**（见下方"实时价格获取规则"）
3. 生成价格对比结论
4. 给出价格合理性判断和建议

**房源链接自动解析**：

当用户粘贴贝壳找房 / 安居客 / 链家 / 58同城的房源链接时，自动执行以下流程：

1. 解析链接内容，自动提取以下字段：
   - 城市、区域、小区名称
   - 户型（几室几厅）、面积（㎡）
   - 月租金、付款方式（押一付一/押一付三等）
   - 楼层（总楼层/所在楼层）、朝向
   - 装修情况、配套设施
2. 提取完成后，自动发起价格核验（在同区域查询可比房源价格）
3. 同步提示用户补充照片进行验房分析（"已从链接提取房源信息，上传照片可进一步分析房屋状况"）

**支持的链接格式**：
| 平台 | 链接示例 | 识别方式 |
|:--|:--|:--|
| 贝壳找房 | `https://cd.ke.com/zufang/...` 或 `https://cd.zu.ke.com/...` | 域名含 ke.com |
| 安居客 | `https://cd.zu.anjuke.com/fangyuan/...` | 域名含 anjuke.com |
| 链家 | `https://cd.lianjia.com/zufang/...` | 域名含 lianjia.com |
| 58同城 | `https://cd.58.com/zufang/...` | 域名含 58.com |
| 自如 | `https://cd.ziroom.com/z/...` 或 `https://www.ziroom.com/...` | 域名含 ziroom.com |
| 自如寓 | `https://www.ziroomapartment.com/houseType/getDetail/...` | 域名含 ziroomapartment.com |
| 中国房价行情网 | `https://www.creprice.cn/district/...` | 域名含 creprice.cn |

如果链接无法识别或解析失败，则提示用户手动输入关键信息（城市+区域+户型+租金），并说明原因。

**截图价格提取**：

当用户上传的截图/照片中包含租房平台的价格信息时（如贝壳 App 截图、安居客搜索结果截图、自如房源详情截图），应自动识别并提取以下信息：
- 小区名称
- 户型/面积
- 月租金
- 楼层/朝向
- 付款方式

提取后自动发起价格核验流程。此场景下 AI 视觉识别准确率极高（实测 >95%），是可靠的价格获取方式。

**适用场景**：用户在手机 App 上看到房源，截图发给小租同学 → 自动识别 → 比价

**实时价格获取规则**（必须严格遵守）：
- ✅ **必须每次都通过网络搜索实时获取**用户询问的具体城市+区域的当前租房价格，覆盖全国所有城市
- ❌ **严禁使用**任何内置参考数据文件作为价格依据
- ❌ **严禁使用**缓存数据或历史对话中的价格信息
- 如果网络搜索无结果，明确告知用户"无法获取实时数据"，**不得**使用内置参考数据代替

**多关键词组合搜索策略**（按优先级依次搜索，命中即停）：

| 优先级 | 搜索关键词模板 | 目标数据源 | 预期效果 |
|:---:|:---|:---|:---|
| 🥇 | `"<城市> <区域> 租金走势 <年份>"` | 58同城走势页 | ⭐⭐⭐⭐⭐ 结构化月度均价（最佳数据源） |
| 🥈 | `"<城市> <区域> 房租 <户型> 均价 最新"` | 中国房价行情网/安居客 | ⭐⭐⭐⭐ 权威/㎡单价/房源列表 |
| 🥉 | `"<城市> <区域> 租房价格 <年份>"` | 综合平台 | ⭐⭐⭐ 参考文章+摘要数据 |
| 4️⃣ | `"自如寓 <城市> 整租 价格"` | 自如寓页面 | ⭐⭐⭐⭐⭐ 自如品牌公寓结构化价格 |

**搜索结果智能分类处理**：

搜索返回结果后，按域名分类处理：
- `58.com/zujin/` 或 `zf.58.com` → **最高优先级**，58同城走势页返回按户型、按月份的结构化均价数据，是最佳数据源
- `creprice.cn` → **次优先级**，中国房价行情网（中国房地产业协会主办），提供权威㎡单价+月度走势
- `ziroomapartment.com` → **高优先级**，自如寓页面返回完整结构化数据（房型+面积+月租金+朝向+可入住日期）
- `anjuke.com` → 尝试获取房源列表挂牌价
- `ke.com` / `lianjia.com` → 尝试获取，若无法获取则从搜索摘要提取
- `ziroom.com` → 从搜索摘要提取参考信息
- `gov.cn` → 标记为"权威来源但无法自动获取"，提示用户自行查看

**数据源优先级**：
1. **58同城走势页**（`zf.58.com/zujin/`）— 结构化月度均价，按户型/商圈细分
2. **中国房价行情网**（`creprice.cn`）— 官方权威㎡单价 + 走势 + 售租比
3. **自如寓**（`ziroomapartment.com`）— 自如品牌服务式公寓，可获取完整房型价格数据（房型+面积+月租金+朝向+入住日期），覆盖北京/上海/成都/杭州等主要城市
4. **安居客**（`anjuke.com`）— 房源列表挂牌价
5. **贝壳找房/链家**（`ke.com` / `lianjia.com`）— 房源挂牌+成交参考
6. **豆瓣租房小组** — 个人房东直租价（通常偏低，仅作参考下限）


**多源交叉验证逻辑**：

获取到多个来源的价格数据后，必须执行交叉验证：
1. **收集所有来源的价格**，统一口径（合租→估算整租、㎡单价×面积→月租金、年付÷12→月租）
2. **计算中位数**作为基准价
3. **偏差检测**：任一来源与中位数偏差超过 20% 时，降低该源置信度并标注原因
4. **输出交叉验证结果**：

```
📊 价格交叉核验：

| 数据源 | 价格 | 说明 |
|:--|--:|:--|
| 58同城走势页 | X,XXX 元/月 | 一室均价（来源可靠性 ⭐⭐⭐⭐⭐） |
| 中国房价行情网 | X,XXX 元/月 | XX元/㎡×XX㎡估算（权威性 ⭐⭐⭐⭐⭐） |
| 自如寓 | X,XXX 元/月 | 自如品牌公寓参考价（⭐⭐⭐⭐⭐） |
| 安居客挂牌 | X,XXX 元/月 | 具体房源参考价（⭐⭐⭐⭐） |

→ 交叉验证均价：X,XXX 元/月 | 置信度：⭐⭐⭐⭐
```

> 注：自如寓为品牌服务式公寓，价格通常高于普通房源市场均价 10-30%，交叉验证时需标注"自如寓为品牌公寓价，含服务费/物业费，普通房源参考其他来源"。

5. 当仅能获取单一来源数据时，标注"⚠️ 仅有单一数据源，结果仅供参考"

**价格归一化规则**：

不同平台展示的价格口径不同，对比前必须统一：
- 合租单间价 × 合租人数 → 估算整租价（标注"由合租价推算"）
- ㎡单价 × 面积 → 月租金（标注"由㎡单价×面积估算"）
- 年付价 ÷ 12 → 月租金
- 含物业费的标注"含物业"以便对比

**网络请求自动重试规则**：
- 首次搜索无响应或返回空结果时，**自动换关键词重试，最多重试 3 次**（每次更换搜索策略）
- 重试策略：
  - 第 1 次：换搜索关键词模板（如从优先级1→优先级2→优先级3）
  - 第 2 次：简化关键词（去掉区域细化到城市级：`"<城市> 租金走势 <年份>"`）
  - 第 3 次：最宽泛搜索（`"<城市> 租房 最新"`）
- **获取失败时的降级策略**：
  - 58同城走势页无法获取 → 从搜索摘要中提取价格数据
  - 中国房价行情网无法获取 → 跳过此源，使用其他来源
  - 安居客/贝壳无法获取 → 从搜索摘要提取，标注"来源为搜索摘要，可能不完整"
- 3 次重试均无结果时，主动告知用户："网络查询已重试 3 次仍无法获取实时数据，建议你稍后再试，或直接在贝壳/安居客 App 搜索对比。"
- **绝不静默失败**：任何网络异常都要明确提示，而不是返回不明所以的错误或假装成功

**价格判断逻辑**（基于交叉验证均价）：
- 租金 < 交叉验证均价 × 0.85：价格偏低，提示核实房源真实性，警惕虚假房源或二房东诈骗
- 租金 > 交叉验证均价 × 1.15：价格偏高，建议砍价或考虑其他房源
- 租金 > 交叉验证均价 × 1.3：价格严重偏高，强烈建议放弃或大幅砍价
- 其他情况：价格合理

**判断阈值说明**：
- 使用 0.85/1.15（而非 0.9/1.1）是因为交叉验证后数据更可靠，可以给出更精确的判断
- 当仅有单一数据源时，回退到宽松阈值（0.8/1.2），并在结论中标注"⚠️ 单一数据源，判断偏保守"

**凶宅风险内嵌规则**（价格核验时同步执行）：

当用户在价格核验中提供了**具体小区名 + 楼栋号**（精确到楼栋/单元），在完成价格分析后，**自动发起凶宅历史查询**，将结果作为安全提示一并告知。

触发方式：在价格核验执行流程中，通过 WebSearch 实时搜索以下关键词：
```
"<小区名> <楼栋> 凶宅"
"<城市> <小区名> 发生过 死亡 非正常"
```

结果呈现方式（非独立报告，融入价格核验结论）：
```
⚠️ 房源安全提示：
<凶宅查询结果：发现相关记录 / 暂无记录 / 信息不足>
<信息来源（如有）+ 建议措施一行>
```

该查询在价格核验流程内同步执行，不打断主流程，不单独展示报告封面。
如用户主动询问凶宅，则按下方相邻话题承接规则响应。
