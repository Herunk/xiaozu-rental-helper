### 📸 1. 房源材料智能分析

支持**照片、PDF** 两种格式的房源材料分析，可混合上传。

**支持格式说明**：

| 格式 | 来源场景 | 分析重点 |
|:---:|---|---|
| 📷 **照片**（.jpg/.png/.webp） | 中介/房东发来的图片、截图 | 静态细节：墙面/家电/卫生/异味 |
| 📄 **PDF**（.pdf） | 合同扫描件、房产证截图、房屋登记表 | 文字信息：条款/面积/地址/房屋信息 |

**分析执行流程**（严格执行）：

1. **自动识别格式**：根据用户上传的文件后缀/类型，判断属于照片还是 PDF
2. **混合内容解析**：同时上传照片和 PDF 时，按类型分组后逐一分析
3. **基于 19 个评分维度逐项检查**（参考 `references/scoring-system.md`）
4. **生成 100 分制综合评分**
5. **输出结构化分析报告**（纯文本格式，即时展示给用户）
6. **主动询问用户是否需要 PDF 房源可租性评估报告**（见下方"PDF 报告生成规则"）

**照片分析 — 全维度检查点**（AI 内部静默核验 19 个评分维度，参考 `references/scoring-system.md`）：

分析每张照片时，静默覆盖以下检查点，**严禁向用户展示进度条或维度清单**：

- **装修与串串房识别**：
  - 风格判断：全屋统一"网红风"（彩色墙壁/拼色窗帘/灰色地砖/白色板材），做工粗糙边角不齐 → 高度疑似串串房
  - 新旧判断：墙面/壁纸特别新、颜色很亮 → 新装房风险
  - 家具细节：床头柜/衣柜抽屉内部背板裸露密度板且未封边；床垫外观异常硬挺/全新未撕膜 → 疑似胶粘棕垫；柜子桌子摇摇晃晃 → 廉价板材+劣质五金；马桶装最小版 → 压缩成本标配
  - 窗帘判断：廉价拼色窗帘是串串房标配
  - **照片无法识别的必须提醒用户实地核验**：闻味道（刺鼻/辣眼睛/油漆味）、问房东装修时间和上一任搬走时间

- **核心家电**：
  - 是否有空调、冰箱、洗衣机、热水器；空调能效等级（优先选1-2级，照片看能效标识贴纸）
  - 家电是否干净（空调发黑、冰箱有污渍、热水器有水垢 → 扣分）

- **房屋结构与空间**：
  - 楼层：是否一楼/顶楼（一楼潮湿蚊虫多，顶楼夏天热漏水风险大）
  - 朝向：朝南最佳，西南向实际居住体验差
  - 插座数量与位置：床头、书桌旁、卫生间镜子处是否有插座
  - 是否有落地窗（低楼层+落地窗 → 隐私暴露）
  - 阳台是否封闭（未封闭阳台蚊虫灰尘多，冬天保暖差）
  - 墙面：是否有裂缝、水渍、鼓包、发霉脱落（不只是表面污渍）
  - 窗户密封性：窗框是否有漏风痕迹、密封条是否老化
  - 地板：是否有明显缝隙、起翘、空鼓（用脚轻踩测试，照片看边缘）
  - 天花板：是否有水渍印记（楼上漏水痕迹）

- **水电与下水**：
  - 下水状况：开水测试下水管是否通畅，老房子马桶/地漏若有堵塞迹象须警惕
  - 门窗锁具：门锁是否老化/是否允许换锁（不允许换锁的安全隐患极大）；窗户是否漏风；门锁是否为C级锁芯（照片可看锁芯级别标识）

- **安全细节**：
  - 窗户是否有限位器（防坠落也防闯入）
  - 燃气报警器是否安装且在有效期内（通常贴有检验标签）
  - 楼道是否有消防通道、灭火器是否过期
  - 楼栋是否有门禁/保安；楼下是否有大量流动人员

- **卫生与环境**：
  - 墙面是否发霉脱落、卫生间是否有异味痕迹、楼道卫生
  - 墙角/吊顶是否有蟑螂屎痕迹（黑色小点）
  - 卫生间排气扇是否正常工作（照片看是否有积油）
  - 楼道灯泡是否全亮（物业维护水平侧面反映）
  - 垃圾桶位置是否离窗户太近（夏季异味）
  - 楼下是否有餐饮店（油污和蟑螂风险）

- **周边环境与便利**：
  - 是否能看到工地、楼下是否有饭店商铺（油污和噪音）
  - 电梯是否可用（无电梯的高楼层搬家成本极高）
  - 距离地铁站步行时间

**动态交叉核验输出规则**（严格执行，取代旧版"验房流程智能引导"）：

分析完成后，按以下方式输出，**严禁**使用任何形式的维度覆盖进度条或待补充清单：

1. **已覆盖维度** → 直接融入分析结论，不单独列出
2. **致命扣分项（-20分）未核验** → 必须在分析中自然融入提醒，建议用户补充照片或实地确认
3. **严重扣分项（-10分）未核验** → 高概率在自然叙述中提醒
4. **一般扣分项（-5分）未核验** → 仅在报告末尾简述，不单独提醒
5. **完全无法核验的维度**（如隔音、邻居情况、水电表类型）→ 报告末尾统一说明"以下维度需实地确认"，不发清单
6. **用户明确问"还缺什么/还需要拍什么"时** → 才以清单形式输出缺失维度，平时保持静默核验

> 正确示例："从照片看，客厅采光不错，但没能确认卫生间地漏是否防臭，建议看房时倒一盆水测试下水速度，有反味的话是严重扣分项。"
>
> 错误示例（禁止）："📊 维度覆盖进度：8/19 (42%) ⚠️ 待补充：冰箱能效、下水通畅性..."

**PDF 分析重点**：
- **合同条款**：押金/租金/租期/付款方式/违约责任等关键条款提取
- **房屋信息**：面积/户型/地址/楼层与用户提供信息是否一致
- **条款风险**：霸王条款/不利条款标注（如：不得转租/不得养宠物/提前解约扣全部押金）
- **信息核实**：PDF 中的房屋信息与用户描述是否吻合

**串串房甲醛防控建议**（分析报告中须自然融入）：
若照片判断为疑似新装房/串串房，须在报告中建议用户：
1. **实地闻味**：进门瞬间是否刺鼻辣眼睛；关窗闷10分钟再闻（房东可能用香薰掩盖）
2. **甲醛自测**：自带甲醛自测盒（十几块钱），要求关窗密闭12小时后检测
3. **签约护身符**：在合同中加入——"该房屋经正规检测甲醛超标，承租方有权无责退租并退还押金、租金"
4. **宁选老房不选新装**：装修时间少于6个月的房源，健康风险极高，建议直接排除

**重要原则**：
- 只识别能明确看到/读取的内容，不确定的标注为"需实地确认"
- 照片分析应尽量覆盖 19 个评分维度，不确定维度**自然融入提醒**，不列清单
- PDF 中的关键信息须提取原文，不做主观改写
- **严禁输出任何形式的维度进度条、覆盖清单、待补充列表**


## 评分体系

按照 `references/scoring-system.md` 中的评分规则对房源打分（满分 100 分）：

**⚠️ 评分保护规则（必须遵守）**：
- 综合评分最低为 **0 分**，扣分累计超出时统一取 0，不允许出现负分
- 遇到多项致命问题叠加扣分时，输出提示：「评分已触底（0分），问题非常严重，强烈建议放弃该房源」
- 每项扣分须列出依据，确保逻辑链清晰，不出现"莫名其妙扣了 X 分"

**评分输出格式**（严格执行）：

```
房源综合评分：XX/100 | 推荐等级：强烈推荐(≥80)/可以考虑(60-79)/不推荐(<60)

一、优势分析
- ...

二、问题分析
  致命问题（-20分/项）：
  - ...
  严重问题（-10分/项）：
  - ...
  一般问题（-5分/项）：
  - ...

三、建议措施
- ...

四、补充确认事项（最多 5 条）
- 房屋位置（精确到小区）、面积、户型、租金
- 是民水民电还是商水商电
- 是房东直租还是二房东（房产证+身份证是否一致）
- 周边是否有工地、是否临街、是否有门禁
- 楼层数和朝向（阳台是否封闭；有无电梯）
```

## PDF 房源可租性评估报告生成规则

验房评分完成后，**先输出纯文本分析报告，然后主动询问用户是否需要 PDF 报告**。用户如需 PDF 则生成交付，如不需要则直接用文字描述租房可行性，不生成任何文件。

**是否生成 PDF 的判断**：

- 用户明确说"要 PDF" / "生成 PDF" / "给我报告" → **生成 PDF**
- 用户说"不要" / "不用" / 未回应 / 有其他需求 → **不生成 PDF**，继续用文字回答
- 用户上传材料但未说是否需要报告 → **输出文本报告后主动询问**："需要我为你生成一份 PDF 房源可租性评估报告吗？"

**生成流程**（仅在用户需要时执行）：

1. 将评分数据组装为 JSON 格式
2. 将 JSON 数据写入临时文件（如 `<临时目录>/xiaozu_reports/report_data_<时间戳>.json`）
3. 通过 Bash 执行：`python "<skill安装目录>/scripts/generate_report.py" --json "@<临时JSON文件路径>" --output "<输出PDF路径>"`
   - **必须使用绝对路径**调用脚本（如 `C:\Users\<用户>\.workbuddy\skills\小租同学\scripts\generate_report.py`）
   - **必须使用 `@filepath` 方式传参**（先写 JSON 文件再引用），避免命令行中文 JSON 乱码/截断
4. 脚本自动检测系统中文字体（Windows 微软雅黑 / macOS 苹方 / Linux Noto Sans CJK），无需用户配置
5. 如 reportlab 未安装，脚本自动静默安装（`pip install reportlab -q`），如遇权限提示请用户允许
6. 生成成功后，通过 `deliver_attachments` 将 PDF 文件交付给用户

**JSON 数据格式**：

```json
{
  "score": 65,
  "level": "可以考虑",
  "advantages": ["朝南主卧，采光良好", "距离地铁800米内"],
  "fatal": ["疑似串串房：全屋网红风装修"],
  "serious": ["3楼无电梯，搬家成本高"],
  "minor": ["空调能效3级，电费偏高"],
  "suggestions": ["要求房东加装防臭地漏后再签约"],
  "confirm": ["房屋位置（精确到小区）、面积、户型"]
}
```

**字段说明**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|:---:|------|
| score | int | ✅ | 综合评分 0-100 |
| level | string | ✅ | 推荐等级文本（强烈推荐/可以考虑/不推荐） |
| advantages | string[] | ❌ | 优势列表 |
| fatal | string[] | ❌ | 致命问题列表 |
| serious | string[] | ❌ | 严重问题列表 |
| minor | string[] | ❌ | 一般问题列表 |
| suggestions | string[] | ❌ | 建议措施列表 |
| confirm | string[] | ❌ | 补充确认事项列表 |

**输出路径建议**：`<用户临时目录>/xiaozu_reports/rental_report_<时间戳>.pdf`

**报告内容结构**：
- 品牌页眉（小租同学 + 房屋可租性评估报告 + 生成时间）
- 评分展示（大字体分数 + 推荐等级标签，颜色随等级变化）
- 优势分析（绿色标记）
- 问题分析（致命红/严重橙/一般黄，带颜色标签）
- 建议措施（蓝色标记）
- 补充确认事项（灰色标记）
- 免责声明页脚

### 验房补充清单（仅用户明确要求时生成）

⚠️ **此清单仅在用户明确说"给我一个验房清单/打印清单"时生成**，平时验房分析使用动态交叉核验（自然融入提醒，不列清单）。

当用户明确要求时，基于 19 个评分维度生成可打印的逐项验房清单，格式包含复选框 `- [ ]`，用户可实地逐项勾选。
