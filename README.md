## 🎯 为什么需要小租同学？

第一次租房的大学生，面临的不是选择太少，而是**信息差太大**：

- 中介说"精装修拎包入住"，实际是甲醛超标的串串房？
- 房东报价 3000/月，你不知道同小区均价只要 2200？
- 合同里藏着"提前解约扣全部押金"，你没看出来就签了？
- 商水商电一个月多花几百块，入住后才发现？

**小租同学**就是帮你消除这些信息差的 AI 助手。它不会替你做决定，但会在你做决定之前，让你**看清楚每一个坑**。

## 📸 功能详情

### 1. 拍照验房 — AI 识坑，19 个维度逐项打分

**支持格式**：照片（.jpg/.png/.webp）+ PDF（合同/房产证），可混合上传

**分析能力**：

| 维度 | 检查内容 |
|---|---|
| 🔴 **串串房识别** | 网红风装修、密度板裸露、胶粘棕垫、廉价五金、新装房甲醛风险 |
| 🏠 **房屋结构** | 楼层、朝向、落地窗、阳台封闭性、插座数量 |
| ❄️ **核心家电** | 空调/冰箱/洗衣机/热水器是否齐全，能效等级 |
| 🔇 **隔音情况** | 隔墙厚度、窗户类型、电器噪音 |
| 🚰 **下水状况** | 管道通畅性、地漏防臭 |
| 🏘️ **周边环境** | 工地、商铺油烟、门禁保安、地铁距离 |
| 📄 **PDF 解析** | 合同条款提取、面积核实、霸王条款标注 |

**评分输出示例**：

```
房源综合评分：65/100 | 推荐等级：可以考虑

一、优势分析
- 朝南主卧，采光良好
- 距离地铁 800 米内

二、问题分析
  致命问题（-20分/项）：
  - 疑似串串房：全屋网红风装修
  严重问题（-10分/项）：
  - 3楼无电梯，搬家成本高
  一般问题（-5分/项）：
  - 空调能效3级，电费偏高

三、建议措施
- 要求房东加装防臭地漏后再签约
- 自带甲醛自测盒，关窗密闭12小时后检测
```

> 评分完成后可选择生成 **房源可租性评估报告**（PDF），带品牌页眉、颜色分级标签，方便保存或分享

### 2. 价格核验 — 实时查价，全国覆盖

**工作方式**：每次查询都通过实时搜索获取最新数据，**不使用任何内置或缓存数据**

```
输入：城市 + 区域 + 面积/户型 + 租金
  ↓
实时搜索：贝壳找房 > 安居客 > 链家 > 58同城 > 豆瓣租房
  ↓
输出：价格对比 + 合理性判断 + 砍价/避坑建议
```

**判断逻辑**：

| 价格 vs 均价 | 判定 | 建议 |
|---|---|---|
| < 均价 × 0.9 | 🔴 偏低 | 警惕虚假房源/二房东诈骗，核实房源真实性 |
| 均价 × 0.9 ~ 1.1 | 🟢 合理 | 结合综合条件做决定 |
| > 均价 × 1.1 | 🟡 偏高 | 尝试砍价或考虑其他房源 |

**内嵌凶宅查询**：当用户提供具体小区名 + 楼栋号时，自动同步查询凶宅历史，融入价格报告一并告知。

## ⚙️ 技术架构

### 数据流

```
用户输入
  ├── 照片/PDF → 格式识别 → 逐项评分 → 评分报告 → [可选] PDF报告
  ├── 价格查询 → WebSearch实时搜索 → 价格对比脚本 → 合理性判断
  └── 问答 → 知识库匹配 → [水电等需实时查询] → 结构化回答
```

### 价格对比引擎

`scripts/price_comparison.py` — 纯计算工具，无内置数据依赖：

```bash
python price_comparison.py \
  --city 深圳 --district 南山区 \
  --house-type 一室一厅 \
  --rent 3500 --avg-price 3800 \
  --min-price 3200 --max-price 4500 \
  --json
```

输出：
```json
{
  "status": "success",
  "price_level": "合理",
  "diff_percent": -7.9,
  "conclusion": "该房源价格合理，与深圳南山区一居室均价（3800元/月）基本一致。",
  "recommendation": "价格合理，建议结合房源综合条件做最终决定。"
}
```

### PDF 报告生成器

`scripts/generate_report.py` — 基于 ReportLab，自动检测系统中文字体：

```bash
python generate_report.py --json "@report_data.json" --output "report.pdf"
```

特性：
- 自动检测 Windows/macOS/Linux 中文字体（微软雅黑/苹方/Noto Sans CJK）
- 如未安装 ReportLab，自动静默安装（`pip install reportlab -q`）
- 支持颜色分级标签（致命红/严重橙/一般黄/优势绿）
- **中文字体注册失败时打印明确警告**（不会静默降级为英文字体导致乱码）

## 📋 核心规则

| 序号 | 规则 | 说明 |
|:---:|---|---|
| 1 | 🔴 **安全底线** | 绝对不推荐二房东的房子，涉及二房东必须明确提示"请尽量避开" |
| 2 | 🏠 **房屋类型** | 优先推荐普通住宅，明确说明商住两用/回迁房/自建房的缺点 |
| 3 | 📍 **选房区域** | 优先推荐公司 3-5 公里内、地铁站附近的小区 |
| 4 | 💰 **价格实时** | 每次价格核验必须通过 WebSearch 实时获取，**严禁使用**内置或缓存数据 |
| 5 | 🔌 **水电实时** | 每次水电燃气查询必须实时搜索用户所在城市价格，**严禁使用**默认城市数据 |
| 6 | 💬 **语言风格** | 使用大学生易懂的口语，避免专业术语 |
| 7 | ⚠️ **信息准确性** | 不确定的信息明确标注，不编造内容 |
| 8 | 🚫 **空值/缺失** | JSON 非必填字段为空时使用默认值（score→0, level→"未评级", 列表→空列表），不中断流程 |
| 9 | 🚫 **超长文本** | 单条文本超 500 字时截断并标注"[已截断]"，防止 PDF 排版溢出 |
| 10 | 🚫 **特殊字符** | 输入中的 `<` `>` `&` `"` `'` 须转义为 HTML 实体后再写入 PDF |
| 11 | 🚫 **非法格式** | 上传非 .jpg/.png/.webp/.pdf 文件时，明确提示"不支持该格式，请上传照片或PDF" |
| 12 | 🔄 **异常处理** | 字体注册失败或 ReportLab 异常时，**打印明确警告**而非静默降级 |

| 6 | 在 PDF 报告中输入 HTML/Markdown 格式内容 | JSON 数据字段应为纯文本，不支持富文本格式 |

## 🙋 常见问题

<details>
<summary><b>Q1：照片很模糊/很少，评分准吗？</b></summary>

评分基于你提供的材料。照片模糊或不足时，缺失维度会标注"建议补充查看"，评分可能偏低。建议上传至少 5 张清晰照片（客厅、卧室、卫生间、厨房、楼道/门口）。
</details>

<details>
<summary><b>Q2：价格核验为什么每次都要重新搜索？</b></summary>

租房价格受季节、政策、供需影响波动很大。为确保准确性，每次都必须通过 WebSearch 实时获取最新数据，不使用任何缓存或内置数据。
</details>

<details>
<summary><b>Q3：PDF 报告里中文显示为方块怎么办？</b></summary>

脚本会自动检测系统中文字体。如果出现方块，说明系统缺少中文字体，请安装：Windows→微软雅黑、macOS→苹方、Linux→Noto Sans CJK。脚本会在字体注册失败时打印明确警告，不会静默降级为英文字体。
</details>

<details>
<summary><b>Q4：支持分析视频中的房源吗？</b></summary>

暂不支持视频分析。请截图后上传照片，或将视频关键帧截图后逐张上传。
</details>

<details>
<summary><b>Q5：凶宅查询靠谱吗？</b></summary>

凶宅查询基于公开网络信息搜索，结果可能不完整。未查到记录不代表绝对安全，建议结合实地走访和邻居交流确认。
</details>

<details>
<summary><b>Q6：上传的合同 PDF 中有个人隐私信息怎么办？</b></summary>

小租同学不会存储或外传任何用户数据。但建议在分析前遮挡身份证号、银行卡号等敏感信息，仅保留合同条款部分。
</details>

<details>
<summary><b>Q7：评分 60 分以下一定不能租吗？</b></summary>

不一定。评分反映的是基于照片可识别的问题严重程度，部分问题（如家电老旧）可通过协商解决。关键看致命问题（如疑似串串房、安全隐患）是否存在。
</details>

<details>
<summary><b>Q8：支持哪些城市？</b></summary>

价格核验和水电查询覆盖全国所有城市，通过实时搜索获取数据，无城市限制。评分维度和流程问答则是通用知识，适用于所有城市。
</details>

<details>
<summary><b>Q9：数据来源可靠吗？</b></summary>
